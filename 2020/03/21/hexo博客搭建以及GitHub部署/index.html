hexo博客搭建以及GitHub部署

Step 1：下载安装nodejs

​ 因为hexo是基于nodejs生成的，所以需要首先安装nodejs。进入nodejs.org网址，下载对应系统下的LTS版本，然后进行安装，默认安装即可。

​ 安装完成后，可以在命令行使用 node -v 查看所安装nodejs版本，使用 npm -v 查看包管理器版本，确认是否安装成功；

​ 另外：建议使用Git Bash

Step 2：安装hexo框架

​ 这里推荐先另行安装Git，然后在任意文件夹中点击右键 Git Bash Here ，进入命令行操作。

​ 在命令行输入下面代码，安装npm的镜像源。

​ npm install -g cnpm –registry=https://registry.npm.taobao.org

​ 安装完成后可使用 cnpm -v 查看版本。

​ 然后使用下面代码安装hexo框架：

​ cnpm install -g hexo-cli

​ 安装完成后使用 hexo -v 查看版本号，确认安装完成。

Step 3：hexo初始化

​ 首先新建一个文件夹用于放置博客的相关文件，然后在此文件夹中右键 Git Bash Here 进入命令行操作。

​ 输入以下代码初始化hexo：

​ hexo init

​ 初始化完成后，可以在文件夹中发现如下内容：

​ 然后输入 hexo s 开始运行，运行启动后会返回用于一个本地查看的链接，默认链接为：

​ http://http://localhost:4000/

​ 可以在浏览器中输入上述网址查看生成的效果。

Step 4：部署hexo到Github的准备工作

​ 首先登陆GitHub，新建一个库，库名需设置为固定格式如下：

​ id.github.io

​ 其中id代表所登陆的Github账号id，如下图所示：

​ 请忽略上述报错信息，正常第一次创建不会出现。创建完成后，便可以得到一个属于你的仓库地址，可用于部署上一步新建的hexo项目。

​ 然后输入下面代码安装Git部署的插件：

​ npm install hexo-deployer-git –save

Step 5：部署hexo到Github

​ 修改文件夹下的配置文件 _config.yml （可以在Git Bash中使用vim修改，也可使用其他文本编辑器进行修改，如Notepad++）。在配置文件最后将部署部分的type和repo修改如下，repo部分就是在上一步中新建的仓库地址，为了规范期间，同时新增了branch。

​ 然后便可以进行部署工作了，在Git Bash中输入下面代码：

​ hexo d

​ 开始推送后，会要求输入GitHub的账号和密码，依次输入即可。

​ 部署完成后，在浏览器中输入上一步的仓库名部分即可访问hexo博客。
